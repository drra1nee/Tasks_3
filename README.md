### Поддерживаемые сущности и контракты

#### Задача (`Task`)
- Представляет задачу платформы с валидацией через дескрипторы
- **Поля:**
  - `id`: уникальный идентификатор (только чтение, авто-генерация)
  - `payload`: описание задачи (строка, 1-500 символов, не пустая)
  - `priority`: приоритет задачи (целое число, 1-10)
  - `status`: статус задачи (`Pending`, `InProgress`, `Done`, `Cancelled`)
  - `created_at`: время создания (только чтение, авто-установка)
- **Вычисляемые свойства(`@property`):**
  - `is_ready`: `True` если статус `Pending` и есть описание
  - `is_completed`: `True` если статус `Done` или `Cancelled`
  - `age`: время с момента создания
- **Методы:**
  - `change_status(new_status)`: смена статуса с проверкой переходов
  - `change_priority(new_priority)`: смена приоритета
  - `to_dict()`: сериализация в словарь
  - `from_dict(data)`: десериализация из словаря
- **Сравнение и хеширование:** на основе `id`

#### Правила переходов статусов

| Текущий статус | Разрешённые переходы |
|----------------|--------------------|
| `Pending` | `Pending`, `InProgress`, `Cancelled` |
| `InProgress` | `InProgress`, `Done`, `Cancelled` |
| `Done` | `Done`  |
| `Cancelled` | `Cancelled` |

---

#### Дескрипторы

| Дескриптор | Тип  | Назначение |
|------------|------|------------|
| `NonEmptyString` | Data | Валидация строки (не пустая, макс. длина) |
| `PriorityDescriptor` | Data | Валидация приоритета (1-10, целое число) |
| `StatusDescriptor` | Data | Валидация статуса и переходов |
| `ReadOnlyDescriptor` | Data | Защита от изменения (id, created_at) |

---

#### Исключения

| Исключение | Описание |
|------------|----------|
| `InvalidDescriptionError` | Некорректное описание задачи |
| `InvalidPriorityError` | Приоритет вне диапазона 1-10 |
| `InvalidStatusError` | Недопустимый статус |
| `TransitionError` | Недопустимый переход между статусами |

#### Контракт источника (`TaskSource`)
- Описан через `typing.Protocol` с декоратором `@runtime_checkable`
- Требует реализации метода:
  - `get_tasks()`: возвращает итератор задач
- Проверка соблюдения контракта:
  - `validate_source(source)`: runtime-проверка через `isinstance`

---

#### Источники задач

**Генератор задач (`GeneratorTaskSource`):**
- Реализует контракт `TaskSource` без явного наследования
- Инициализируется количеством задач (`count`)
- Генерирует задачи со случайными описаниями из предустановленного списка:
  - `"Process order"`, `"Send notification"`, `"Check status"`, ...
- Приоритет: случайное число 1-10
- Статус: случайный `Pending` или `InProgress`
- Функция `create_generator_source(count)`: упрощает создание

**API-заглушка (`APITaskSource`):**
- Реализует контракт `TaskSource` без явного наследования
- Имитирует внешний источник задач (API)
- Инициализируется:
  - `tasks_data`: список словарей с данными задач
  - Если данные не переданы - используются данные по умолчанию
- Извлекает `priority` и `status` из данных (или использует значения по умолчанию)
- Преобразует dict в строковый `payload`
- Методы:
  - `get_tasks()`: возвращает итератор задач
  - `refresh()`: имитирует обновление данных из API
- Функция `create_api_source(tasks_count)`: упрощает создание

#### Очередь задач (`TaskQueue`)
- Коллекция задач с поддержкой итерации, фильтрации и потоковой обработки
- Совместима со стандартными конструкциями Python (`for`, `list`, `sum`, `sorted`)

**Базовые методы:**

| Метод | Описание |
|-------|----------|
| `add_task(task)` | Добавить задачу в очередь |
| `remove_task(task_id)` | Удалить задачу по ID (возвращает `bool`) |
| `get_task_by_id(task_id)` | Получить задачу по ID или `None` |
| `__len__()` | Количество задач (`len(queue)`) |
| `__iter__()` | Итератор (поддержка `for`, `list()`) |
| `__contains__()` | Проверка наличия (`task in queue`) |

**Ленивые фильтры (генераторы с `yield`):**

| Метод | Описание |
|-------|----------|
| `filter_by_status(status)` | Задачи с указанным статусом |
| `filter_by_priority(min, max)` | Задачи в диапазоне приоритетов |
| `filter_ready()` | Готовые к выполнению задачи |
| `filter_completed()` | Завершённые задачи |

**Потоковая обработка:**

| Метод | Описание |
|-------|----------|
| `batch(size)` | Пакеты задач указанного размера |
| `take(n)` | Первые `n` задач |
| `skip(n)` | Пропустить первые `n` задач |

**Агрегатные методы:**

| Метод | Описание |
|-------|----------|
| `total_priority()` | Сумма приоритетов всех задач |
| `max_priority()` | Максимальный приоритет |
| `min_priority()` | Минимальный приоритет |

**Примеры использования:**

```python
from src.queue.task_queue import TaskQueue
from src.models.task import Task

queue = TaskQueue()
queue.add_task(Task(payload="Order", priority=7, status="Pending"))
queue.add_task(Task(payload="Notify", priority=3, status="Done"))

# Итерация и фильтрация
for task in queue.filter_by_status("Pending"):
    print(task.payload)

# Потоковая обработка
for batch in queue.batch(100):
    process(batch)

# Пагинация
from itertools import islice
page = list(islice(queue.skip(10), 5))

# Агрегация
total = queue.total_priority()
```

### Особенности реализации

**Особенности реализации:**
  - Все дескрипторы имеют `__get__`, `__set__`, `__delete__`, то есть они `Data descriptors`
  - `Non-Data descriptors` не используются
  - Хранение значений через `instance.__dict__`
  - Защита: `ReadOnlyDescriptor.__set__` разрешает первую установку, запрещает повторную

- **Хранение данных:**
  - Значения хранятся в `instance.__dict__`
  - Автоматическая очистка при удалении объекта (нет утечек памяти)
  - Нет проблем с переиспользованием `id()` объектов

- **Property для вычисляемых свойств:**
  - Встроенный декоратор `@property` для `is_ready`, `is_completed`, `age`
  - Read-only доступ (без `@setter`) — защита от изменения
  - Вычисление значения при каждом обращении

- **Контрактное программирование:**
  - Источники задач не наследуют общий базовый класс
  - Используется `typing.Protocol` для статической проверки типов

- **Типизация:**
  - Все публичные интерфейсы имеют аннотации типов

- **Инкапсуляция:**
  - Публичный API через дескрипторы
  - Внутреннее хранение через `__dict__` с префиксом `_` для read-only полей

- **Очередь задач (TaskQueue):**
  - Все фильтры - генераторы (`yield`), данные не копируются в память
  - Каждый вызов `iter(queue)` создаёт новый итератор (поддержка повторного обхода)
  - Корректная работа с большими объёмами (10 000+ задач)
  - Эффективность: O(1) для `add`, O(n) для `remove`, O(1) для итерации
